# Backlog Task Quality Checklist

Use this checklist when creating or reviewing a task in `backlog/tasks/`.

## When to use

- Before setting a new task to `todo`
- Before moving a task from draft notes into `backlog/tasks/`
- During human review of planner-generated tasks

## Checklist

- [ ] **Objective is clear:** the task states a concrete outcome, not a vague improvement.
- [ ] **Scope is bounded:** in-scope changes are explicit and realistically small.
- [ ] **Out of scope is explicit:** boundaries prevent accidental scope expansion.
- [ ] **Acceptance criteria are testable:** each criterion can be verified as pass/fail.
- [ ] **Dependencies are correct:** prerequisite tasks are listed and actually required.
- [ ] **Priority is justified:** priority reflects MVP impact, not personal preference.
- [ ] **Implementation notes are practical:** notes help execution without over-specifying solution details.
- [ ] **File naming is valid:** filename follows `TASK-###-description.md`.
- [ ] **Status is valid:** status is one of `todo`, `in-progress`, `done`, or `blocked`.
